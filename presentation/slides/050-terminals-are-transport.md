Terminals are great for scripting and remote access.

Bad at rich user interfaces.

# Translation

ターミナルはスクリプトや遠隔操作には強い。
リッチなUIには弱い。

---

Here is the thing, though. The terminal is not *bad*...

If you want to script something, run it remotely, pipe it to another program, automate it — the terminal is perfect. It is built for exactly that.

Where it falls down is when you want to present a *rich* user interface. The moment you want layout, fonts, images, anything more than text on a grid — that is where it starts to show it's age.
